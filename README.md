- Проектирование базы данных туристического агентства
Данный проект демонстрирует комплексный подход к созданию базы данных для автоматизации работы туристического агентства. В рамках работы были выполнены следующие этапы:

1. Описание предметной области
Определены основные бизнес-процессы (бронирование туров, консультации, работа с клиентами и туроператорами).

Выявлены стейкхолдеры: менеджеры, руководитель, администратор, клиенты, туроператоры.

Сформулированы количественные и качественные цели автоматизации.

Описаны используемые классификаторы (ОКВЭД, ОКАТО, ОКВ, ОКИН и др.).

2. Формулирование требований
Составлены User Stories для всех ролей.

Определены входные и выходные потоки данных для каждого актора.

Выявлены текущие проблемы ручного учёта и сформулированы требования к будущей системе.

3. Выделение сущностей и атрибутов
Проведена декомпозиция предметной области на сущности (справочники, объекты, документы).

Для каждой сущности определён атрибутивный состав с указанием типов данных.

Прогнозируемые объёмы данных помогают оценить масштаб системы.

4. Построение концептуальной модели (ER-диаграмма)
Использована нотация Чена.

Выделены сильные и слабые сущности, идентифицирующие связи.

Классифицированы атрибуты: простые, составные, многозначные, производные, обязательные и необязательные.

Визуализация выполнена в Draw.io.

5. Логическое проектирование (реляционная модель)
Переход к нотации Мартина (Crow's Foot).

Нормализация данных, разрешение связей «многие ко многим».

Введение первичных (PK) и внешних (FK) ключей.

Обеспечение ссылочной целостности.

6. Физическое проектирование (DDL-скрипты)
Разработаны скрипты для PostgreSQL.

Использованы современные стандарты (SQL:2003 Identity) вместо устаревших SERIAL.

Внедрены пользовательские типы данных (DOMAIN, TYPE) для валидации (ФИО, паспорт, адрес).

Настроены ограничения CHECK, стратегии ON DELETE / ON UPDATE.

Построена иерархия DDL для создания БД «с нуля».

7. Реализация и наполнение данными
Развёртывание локального сервера PostgreSQL и настройка DBeaver.

Исполнение DDL-скриптов, создание структуры БД.

Пакетное заполнение таблиц тестовыми данными (INSERT).

Написание DML-запросов для проверки и визуализации данных.

8. Аналитические запросы (на примере зоомагазина)
Дополнительный блок задач по работе с данными (выборка, фильтрация, группировка, соединения).

Закрепление навыков SQL на реальных операциях с таблицами.

- Используемые технологии
Инструмент / Технология	Назначение
PostgreSQL	Реляционная СУБД
DBeaver	Графический клиент для работы с БД
Draw.io	Визуализация ER-диаграмм
SQL (DDL, DML)	Создание структуры и наполнение данными
ОКВЭД, ОКАТО, ОКИН и др.	Классификаторы для стандартизации данных

- ER-диаграммы (пример)
Концептуальная модель (нотация Чена) — отображает сущности, атрибуты и связи на высоком уровне.

Логическая модель (нотация Мартина) — детализированная реляционная схема с PK/FK.

- Результаты
Разработана полноценная реляционная база данных для турагентства.

Реализована логика бронирования, продаж, работы с клиентами и туроператорами.

Обеспечена ссылочная целостность и бизнес-ограничения на уровне СУБД.

Подготовлены аналитические запросы для отчётности и контроля.
